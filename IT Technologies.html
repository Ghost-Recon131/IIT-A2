What can be done now? - MFA
Increase usage of multi factor authentication (MFA) alongside existing username and password. MFA requires an additional code or prompt that the user needs to provide before access is granted, thus providing some protection against stolen login details. MFA can be implemented via push notification over an app on the user's phone such as Microsoft authenticator, sent over SMS or email as well as TOTP tokens generated by apps like Authy (Bojan Jovanovic, 2022). More advanced MFA uses biometrics to verify the user, such as Services Australia, which allows users to create a voiceprint to verify their identity (servicesaustralia, 2021).
FIDO 2 Passwordless authentication
Hardware authentication via WebAuthen / U2F provides an addition level of security, as users need to physically interact with the hardware module, attacks via remote desktop connection is no longer possible. FIDO 2 is a new password-less protocol that skips the need for password entirely, using just a hardware module to perform the cryptographic functions to verify a user. This process is based on the domain of the website they signed up and can't be spoofed by using phishing websites. (Kensington, 2021 and FIDO Alliance, no date).
Zero trust Model
Zero trust means that the organisation's infrastructure is designed to no trust anyone operating inside (local network) or outside the network. It combines the use of MFA, file encryption, access control and Least-privilege access (Yubico, no date). Access control is generally implemented via role-based access and goes along with Least-privilege access which focuses on providing the minimal number of permissions for users to complete their tasks. ie HR department worker can access employee files and make changes on their position and pay, however cannot view their bank details which is restricted to finance department. Another important part of Zero trust is log all data. Such as logging when a user accessed what file, from which device and on what network. Logs are then stored and can be used to find suspicious activity or attempted attacks.
Emerging technologies over the next 3 years (Pye, 2022) - AI / ML threat detection
ML based detection is being tried in Cybersecurity tasks now and is likely to get more usage in the next few years. AI based detecting will mainly benefit against attacks using 0-day exploits, as written in Google’s report (Maddie Stone and Clement Lecigne, 2021), there has been a consistent increase in 0-Day vulnerabilities detected following 2018. AI based techniques will use behavioural analysis discussed in (Shaukat et al., 2020) to determine threats, thus able to react in real time, potentially preventing the 0 day from being exploited until a patch is issued by developers.
Blockchain security
An example of using Blockchain technology can used to decentralise DNS’s which can prevent DDoS attacks targeting DNS servers to stop users from connecting to websites. Currently two Blockchain based DNS solutions already exist Namecoin and Blockstack, where “Each system node can act as a DNS server, where users can perform domain name registration, transmission, and data revision. The decentralization of domain name services makes it impossible for intruders to manipulate or steal the central record.” (Yassine Maleh, 2020)
Blockchain can also be used to verify ownership of a document or certification. Borrowing the idea from NFTs, documents such as contracts, legal agreements, student transcripts can prove that their file is authentic, and they own said file. (Jessica Groopman, 2022)
Government legislation
he EU Cybersecurity Act aims to enforce better information sharing between companies and end users, as well as mandate and standardise some cybersecurity features (Devin Partida, 2022). While these security standards are not yet mandatory, there will help everyone if all businesses operating inside the EU were forced to comply with the new laws. The American ‘Strengthening American Cybersecurity Act’ aims to keep organizations honest to their customers by mandating the report of cyberattacks and ransomware payments they have made (Greig, 2022). Potentially stopping any organisations from trying to hide they were compromised and leave users in the dark about having their information stolen.

Clouds, services, servers
What does it do?
Clouds, services, servers are network clouds that break down huge computational data processing procedures into countless smaller procedures, which are processed and analysed by a system of multiple servers to obtain results and return them to the user. The level of this new technology allows the processing of tens of thousands of pieces of data in a matter of seconds, resulting in a powerful network service. As a result, the use of cloud services can reduce costs, i.e. reduce the cost and effort of purchasing, installing and configuring local infrastructure, and have greater flexibility, i.e. start using the relative applications very quickly without having to send requests and wait weeks for a response. Combining the most resources to improve network competitiveness, i.e. the ability to leverage the cloud provider's global network to bring applications closer to users from all over the world. For the time being, cloud services can offer three types of services, namely IaaS (Infrastructure as a service), PaaS (Platform as a service) and SaaS (Software as a Service). In IaaS services, user does not have permission to manage and access the underlying infrastructure, such as servers, switches, hard disks, etc., but has the right to manage the operating system, and storage content, to install and manage applications, and even to manage network components. In simple terms, if you use IaaS services, then it means that you do not need to sell your own servers, but you still need to install your own server software. Common IaaS services are virtual machines, virtual networking, and storage. PaaS gives users the ability to create, develop and deploy applications on the associated infrastructure using programming languages, libraries, services and development tools that are supported by the cloud service provider. Users do not need to manage the underlying infrastructure, including networks, servers, operating systems or storage. They can only control the application deployed on the operating system in the infrastructure and configure the configurable parameters of the environment in which the application is hosted. In other words, with PaaS services, it means that you neither need to buy a server nor install your own server software, but only need to develop your own website programs. Common PaaS services are database services, web applications and container services. SaaS gives users the ability to use applications from cloud service providers running on a cloud infrastructure. Applications can be accessed from a variety of client devices through a lightweight client interface or programmatic interface. The user does not need to manage or control the underlying cloud infrastructure, including the network, servers, operating system, storage or even individual application functions. If you take this a step further and purchase the services of some online forums or online shops, this means that you don't have to develop your own website, you just use the programs they have developed and they take care of the upgrades, maintenance, adding servers, etc. and you just need to concentrate on running them, like Gmail, Dropbox, etc.

Autonomous Vehicles
What does it do?
Autonomous vehicles or simply self-driving cars are vehicles that can sense their surrounding environment which therefore allows them to function without the need for human interaction. The amount of control car itself has been defined in a 6 levels, labelled as the “6 levels of driving automation”, by “the Society of Automotive Engineers”, these levels range from fully manual which is level 0 all the way to level 5 which is known as full automation (Synopsis, 2022) They are an ever growing area of technology which even during the pandemic had a rise in sales of 78% in 2020 from 2019, with North America being forecasted to lead the market. This is due to them being a pioneer in the field of autonomous vehicles industry, their already established automotive companies and due to it being the home of the biggest technology giants such as Google and Apple (ReportLInker, 2021). The current situation of autonomous vehicles is that even with billions invested into the field and major advancements made, there are still no companies in the market which can provide a fully self-driving vehicle without a human overseer under any conditions (Kasilkowski, 2019). The market does however seem to be full of cars which have at least at level 1 of 6 of the automation level like cruise control and there are also cars in level 2 like Tesla and even level 3 which Honda and Mercedes are pushing forward with. However, this does also bring about the issue of the lack of knowledge of consumers or the lack of information or misinformation provided by companies, for example the controversy surrounding Tesla about their almost false marketing of the Tesla being full self-driving even though their cars are only considered level 2. This has therefore caused fatal crashes and at least 20 deaths (Day, 2021). In terms of the future of autonomous vehicles, it is important to recognise that they are not only desired so they can be used by humans on roads to get to places without driving, but they are also being invested upon by both the government and industry sectors so that they can be utilized in other areas. These areas involve the likes of search and rescue supply convoys or even urban reconnaissance (Connelly et al, 2006). The vision that experts are aiming towards is completely driverless vehicles whether that be for personal use or within the industry, essentially aiming for cars that can be deployed and used anywhere around the world. Getting to this point however is far from easy due to the multitude of hurdles which must first be overcome, the biggest hurdle currently getting these cars to operate safely and effectively in a complex and unpredictable human environment as these self-driving vehicles will have to cope with humans that speed, or those that break the road rules. To address this issue to current environment and or technology being used is a facility at the University of Michigan which is a testing ground for autonomous vehicles which includes traffic infrastructure, underpasses, different terrains like sidewalks and railway tracks. Different scenarios are being tested and a variety of people from different disciplines like psychology are being brought in to home in on the human to machine interaction as there are nearly endless scenarios in which a fully autonomous vehicle will need to operate in (Cusack, 2021). These vehicles run on the design sense, plan and then act, which must be operated effectively and efficiently for these cars to achieve the goals of experts who want safer journeys for drivers as research shows 9 out of 10 crashes occur due to human error, forecasts now have predicted that by 2030 1 in 10 vehicles will be fully autonomous (Kadry, 2021)
What is the likely impact?
As the technology behind autonomous vehicles continues to evolve and adapt, one of the primary markets that will be affected is the automotive industry. The auto industry is currently one of the largest and well renowned industries, valued at around $2.7 trillion worldwide (Mathilde Carlier,2022). As the auto companies, one of the most important priorities is that their product needs to have better and newer features to ensure that they retain and attract consumers. Currently the use of self-driving features is still in its infancy and regarded more as assisted driving. However, as the technology continues to improve companies will have to adapt and implement autonomous driving into their newer products. For those who are slow to acclimate to the new development, it could potentially affect their bottom line as they will not have the same features in comparison to other companies. The continued advancement of autonomous technology could also bring into question the necessity of owning a vehicle once it reaches a point where cars are driverless. Once the level of full autonomous vehicles in which a driver is no longer required is reached, a new market of driverless ride sharing could be introduced. This could potentially affect both the auto industry and the ride sharing industry. For the auto industry, people may no longer have the desire to purchase a privately owned vehicle if there is a more accessible and affordable option. Driverless ride sharing could be more affordable than current ride sharing options due to not having to pay for labour costs. In turn, unemployment rates for people working for ride sharing companies would go up as the companies would most likely opt in for self-driving vehicles to circumvent the labour costs.

Small computing devices
What is the state of the art of this new technology? What can be done now? What is it likely to be able to do be done soon (next 3 years) What technological or other developments make this possible
There are many computing devices that take an input, then do a result or instruction based on that input. Advances in technology have allowed these devices to be smaller, and more easier to understand and use. Along with common devices like phones, calculators and cameras that have set uses, there are other devices like Raspberry Pi, or Arduino, that are open-source and allow the user to build their own digital devices.
Phones continue to access and do more things, from searching up on the internet to find information, to contactless payment with credit cards. Contactless payment is itself a computing device that has helped the way people pay for things, shortening the payment procedure for efficiency and ease. With the usage of Bluetooth, phones are now able to connect to wireless earphones, and with assistance from satellites, can be tracked to locate your phone location in an area, or to contact others.
Devices like Arduinos and Raspberry Pis, which allow the user to create their own devices, interact with buttons, LEDs, motors, speakers, cameras and/or the internet, allowing them to create a device with a purpose specific for the creator’s need.
One trend that has appeared is the usage of AR (Augmented Reality), interacting with the real-world using a phone or other device. Some current uses include displaying information of a product or item with the camera, using the camera to access features in maps, and scanning a room or person to create an accurate, life-sized 3D model for design and media. One example included using phones and flying drones to scan an abandoned town and preserve its environment in 3D (Corridor Crew 2021). With the use of these devices, organisations have been able to create machines that are able to repeat multiple actions and instructions, automating a process or procedure without restrictions such as repetition or imperfection. This has allowed companies to not risk a worker’s health due to handling dangerous tasks such as construction and manufacturing, while also becoming more efficient in the workplace. Some companies have departments that are completely automated, needing only one or two workers to check on their condition, as the automated machines complete their task without error for long periods of time. This automation is not only limited to work environments, and can also be found in homes, using phones or virtual assistants to interact with the surroundings. Google Glass Enterprise is a device used in professional settings that use "visual, audio and location-based inputs" (TechTarget 2019, para. 2) to send and show information. with an internet and Bluetooth connection, users can take photos and videos, search up information, and send information to others without having the need to look away to grab their phone, allowing them to be efficient in the workplace. Although Glass Enterprise "only offers a limited range of features compared to its competitors" (VR-Expert 2021, Sect. 15), its lightweight design paired with its cheap price make it an ideal device in workplaces. As components such as microphones, cameras and CPUs continue to get cheaper, smaller and more powerful, this technology may not only become common in varying workplaces but may also allow for the everyday consumer to use these devices to improve their lifestyles and become more efficient in their use of technology.
With continuing developments in Internet of Things (IoT), some household machines and items have small computer chips capable of sending data, processing and displaying information with the internet (ZDNet 2020) to have extra functionality. Consumers are now able to set reminders, watch videos, send messages and monitor the house through built-in software, without having to rely on their phones or computers. With the use of in-built sensors in these devices, homes become ‘simpler, better, [and] more accessible’ (Integrated Technologies Australia n.d.), allowing consumers to automate household appliances such as the brightness of the lights at night, the thermometer in a room full of people, and/or the time a television or shutter activates based on the user's sleep schedule. As the concept of IoT becomes introduced to more appliances, soon people will be able to control every device in their house through their phone, or with the assistance of sensors, letting them focus on work or hobbies while common tasks are automated.
As technology continues to advance and become more accessible to people, actions and tasks become simpler and more streamlined, assisting people in their daily life. In the future, current computing devices will become more compact, able to store and process more information for cheaper, allowing more people to use technology, while also helping people that are knowledgeable about these devices to be more efficient with them. One current idea that people are trying to advance in is the use of AR and VR (Virtual Reality), with projects like Meta and AR Glasses currently being worked on.
